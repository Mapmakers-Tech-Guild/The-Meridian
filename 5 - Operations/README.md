# 5 — Operations

**Inspiration:** org KBs (e.g. company `03-Ops\knowledgebase`) separate **day-to-day operations** from housekeeping templates and from **archive** cold storage. This area is the Mapmakers **guild ops** home: SOPs, “how we run the guild,” sync rituals, checklists, incident notes that are **durable** but not the same as **3 - Knowledge** patterns (those are more evergreen *ideas*).

| Suggested (add as you go) | Purpose |
| --- | --- |
| `runbooks/` | Repeatable procedures (onboarding a repo, cut a release) |
| `rituals.md` or `cadence.md` | Weekly / monthly guild rhythms |
| `incidents/` | Blameless writeups (if you do them) |

**Not** the same as **0 - Housekeeping** (nav, templates, Egregore) or **4 - Guild/ops** (agent persona packs) — *operations* = how humans run the *organization*.
