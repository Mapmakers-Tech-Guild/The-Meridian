# How to use this open knowledge base

1. **Do not** put this folder in the Egregore `memory` symlink. The **open** KB lives in **a private work area** (this path) or, when you are ready, a **public** repository that mirrors or replaces it.
2. Read [EGREGORE-VS-OPEN-KB.md](../EGREGORE-VS-OPEN-KB.md) so you never point tools at the wrong place. To **pull** open-KB content *into* Egregore, follow [EGREGORE-INTEROP.md](../EGREGORE-INTEROP.md) and keep `the-mapper-s-ego-memory` on the **standard** tree only.
3. For **publishing** workflow and what branches can and cannot hide, read [MEMBRANES-BRANCHES-AND-PUBLISHING.md](../MEMBRANES-BRANCHES-AND-PUBLISHING.md).
4. Add or edit `knowledge/patterns/`, `people/personas/`, and `projects/` for material you are willing to be **public or semipublic** after redaction.

Egregore commands (`/handoff`, etc.) should keep writing to **`the-mapper-s-ego-memory`**, not here.
